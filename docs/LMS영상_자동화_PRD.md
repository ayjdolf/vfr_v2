# LMS 영상 관리 자동화 PRD

> 작성일: 2026-05-18  
> 대상 프로젝트: vfr_v2 (LMS 영상 재생 검수 시스템)

---

## 배경 및 목적

현재 영상 검수는 파일을 로컬로 다운로드한 뒤 vfr_v2에 넣어 수동으로 진행.  
CDN(NCP Global Edge)에 올라간 파일을 URL만으로 직접 검수할 수 있음을 확인,  
이를 확장해 **다운로드 → 변환 → 재업로드 → 퍼지**까지 일괄 자동화를 목표로 한다.

---

## 현재 구현 상태

| 기능 | 상태 | 비고 |
|------|------|------|
| 로컬 파일 검수 | ✅ 완료 | 수동 검수 탭 |
| 로컬 파일 변환 | ✅ 완료 | 수동 검수 탭 → 변환 |
| CDN URL 검수 | ✅ 완료 | URL 검수 탭 (별도) |
| CDN 파일 다운로드 | ✅ 완료 | URL 검수 탭 → 다운로드 버튼 |
| 변환 후 재업로드 | ⬜ 미구현 | NCP 자격증명 필요 |
| 재업로드 후 퍼지 | ⬜ 미구현 | ncp-purge 연동 필요 |

---

## 현재 문제점

### 탭 분리로 인한 중복 기록 문제
URL 검수 후 다운로드 → 수동 검수 탭에서 재검수 시 동일 파일이 Excel 통합 시트에 2회 기록됨.

```
URL 검수  → "URL검수" 시트 저장 + "통합" 시트 저장  ← 1회
     ↓ 다운로드
수동 검수 → "수동검수" 시트 저장 + "통합" 시트 저장 ← 2회 (중복)
```

결과적으로 보고서에서 동일 파일이 2건으로 집계되어 문제 비율 왜곡 발생.

---

## 목표 워크플로우

```
LCMS에서 CDN 경로 확인
        ↓
[통합 검수 탭] CDN URL 입력 → 검수 결과 확인
        ↓ 문제 있을 경우
[다운로드 & 변환] 버튼 한 번으로 처리 → 출력영상/_fix.mp4
        ↓ (향후)
[업로드] NCP Object Storage 동일 경로에 덮어쓰기
        ↓
[퍼지] NCP CDN 캐시 자동 Purge
        ↓
완료
```

### 개편 핵심 변경점
| 항목 | 현재 | 변경 후 |
|------|------|---------|
| URL 검수 위치 | 별도 탭 | 통합 검수 탭 상단 입력줄 |
| URL 파일 변환 | 수동 탭으로 이동 필요 | 결과 행에서 [다운로드&변환] 바로 처리 |
| Excel 저장 | URL/수동 각각 + 통합 (중복) | 통합 시트 하나로만 |
| 중복 기록 | 발생 | 없음 |

> ⚠️ UI 개편은 기존 수동 검수 탭 사용 흐름에 영향이 있어 별도 검토 후 진행

## 요구사항

### 기능 요구사항

#### F1. 통합 검수 탭 (UI 개편)
- URL 입력 + 로컬 폴더 스캔을 하나의 탭에서 처리
- 결과 테이블 통합 (소스 구분 컬럼 추가: URL / 로컬)
- Excel 중복 저장 문제 해결

#### F2. URL 파일 다운로드 & 변환 통합
- URL 검수 결과 행에서 [다운로드 & 변환] 버튼 한 번으로 처리
- 다운로드 → 변환 → 출력영상/_fix.mp4 저장까지 자동
- 진행 상태 표시 (다운로드 %  →  변환 중  →  완료)

#### F3. 변환 후 업로드
- 변환 완료 파일을 원본 CDN 경로에 재업로드
- 업로드 경로는 원본 URL에서 자동 추출 (덮어쓰기)
- 업로드 진행률 표시

#### F4. 업로드 후 자동 퍼지
- 업로드 완료 시 해당 경로 1단계(디렉토리) 퍼지 자동 실행
- 퍼지 결과(성공/실패) UI 표시

#### F5. NCP 자격증명 관리
- Access Key / Secret Key 설정 UI
- 자격증명 로컬 암호화 저장
