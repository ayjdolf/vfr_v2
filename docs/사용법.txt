  LMS 영상 재생 검수 시스템 (vfr_v2) - 사용법
  버전: 2.x  |  작성일: 2026-05-18
================================================================================

--------------------------------------------------------------------------------
■ 시작하기
--------------------------------------------------------------------------------

1. app.py 를 실행합니다.
   - 터미널에서: python app.py
   - 또는 run.bat 더블클릭

2. 데스크톱 창이 뜨면 준비 완료입니다.
   ※ index.html 을 브라우저에서 직접 열면 작동하지 않습니다.
     반드시 app.py 를 통해 실행해야 합니다.

--------------------------------------------------------------------------------
■ 탭 구성
--------------------------------------------------------------------------------

  [📋 수동 검수]   로컬 MP4 파일 폴더 스캔
  [🔗 URL 검수]    CDN URL 로 직접 검수 + 다운로드
  [👁 자동 감시]   지정 폴더 신규 파일 자동 감시
  [📊 보고서]      검수 결과 통계 및 Excel 내보내기

--------------------------------------------------------------------------------
■ 수동 검수 탭 사용법
--------------------------------------------------------------------------------

1. [입력폴더 선택] → 검수할 MP4 파일이 있는 폴더 선택
2. [출력폴더 선택] → 변환된 파일을 저장할 폴더 선택
   (기본값: 프로그램 폴더 내 "출력영상" 폴더)
3. [스캔 시작] → 입력폴더의 모든 MP4 자동 검수

4. 결과 테이블 확인
   - "정상" : 재생 문제 없음
   - "문제" : 아래 항목 중 하나 이상 감지

5. 문제 파일 변환
   - 체크박스로 파일 선택 (또는 [전체선택])
   - [선택파일 변환] 클릭
   - 출력폴더에 {원본파일명}_fix.mp4 로 저장됨

--------------------------------------------------------------------------------
■ URL 검수 탭 사용법
--------------------------------------------------------------------------------

1. CDN URL 입력란에 MP4 주소 붙여넣기
   예) https://fkz4vab0536.edge.naverncp.com/2026/10/BA1003/1630027/BA100301/1/BA100301_1_1.mp4

2. [검수] 클릭 → 결과 표시 (다운로드 없이 직접 검수)

3. 문제가 있는 경우 → [⬇ 다운로드] 클릭
   - 파일이 "입력영상" 폴더에 저장됩니다.
   - 저장 후 수동 검수 탭에서 변환 가능합니다.

※ URL 검수 시 생략 항목
   - DTS 역전 검사: 네트워크 부하로 인해 URL 검수 시 건너뜀
   - Faststart: Range 요청(앞 200KB)으로 moov 위치만 확인

--------------------------------------------------------------------------------
■ 자동 감시 탭 사용법
--------------------------------------------------------------------------------

1. 감시 폴더 경로 확인 (기본: \\ds112.kcu.ac\kcubackup\UPLOADLIST\2026\10)
2. [감시 시작] 클릭 → 5초 간격으로 폴더 모니터링
3. 새 MP4 파일 감지 시 자동 검수 → 결과 기록
4. [감시 중지] 로 종료

--------------------------------------------------------------------------------
■ 보고서 탭 사용법
--------------------------------------------------------------------------------

- 통계 카드: 전체 / 정상 / 문제 건수 확인
- [Excel 보고서 열기]: 검수결과.xlsx 파일 열기
- [TXT 보고서 내보내기]: 텍스트 보고서 저장

Excel 파일 구조:
  - "수동검수" 시트: 수동 검수 결과
  - "URL검수"  시트: URL 검수 결과
  - "자동감시" 시트: 자동 감시 결과
  - "통합"     시트: 전체 결과 통합

--------------------------------------------------------------------------------
■ 검수 항목 설명
--------------------------------------------------------------------------------

  항목          | 내용
  --------------|-------------------------------------------------------------
  VFR           | 가변 프레임레이트 여부 (문제 시 LMS 재생 오류 가능)
  Faststart     | MP4 moov atom 위치 (앞에 있어야 스트리밍 정상)
  DTS 역전      | 타임스탬프 역전 여부 (로컬 파일만 검사)
  오디오 스트림  | 오디오 트랙 존재 여부
  해상도         | 영상 가로x세로 (픽셀)
  재생시간       | 총 길이 (초)

--------------------------------------------------------------------------------
■ 파일 구조
--------------------------------------------------------------------------------

  vfr_v2/
  ├── app.py          ← 메인 실행 파일
  ├── checker.py      ← 검수 로직 (ffprobe/ffmpeg 호출)
  ├── index.html      ← UI
  ├── 입력영상/        ← 로컬 검수용 MP4 + URL 다운로드 저장 위치
  ├── 출력영상/        ← 변환 완료 파일 저장 위치
  ├── 검수보고서/      ← Excel / TXT 보고서 저장
  └── docs/           ← 이 문서 포함 기술 문서

--------------------------------------------------------------------------------
■ 요구사항
--------------------------------------------------------------------------------

  - Python 3.9 이상
  - ffprobe / ffmpeg  (PATH에 등록 또는 bin/ 폴더 배치)
  - pywebview         (pip install pywebview)
  - openpyxl          (pip install openpyxl)

--------------------------------------------------------------------------------
■ 알려진 제한 사항
--------------------------------------------------------------------------------

1. URL 검수 후 [다운로드] → 수동 검수 탭에서 재검수 시
   Excel 통합 시트에 동일 파일이 2건 기록될 수 있습니다.
   (향후 통합 검수 탭으로 개편하여 해결 예정)

2. URL 검수는 DTS 역전 검사를 생략합니다.
   정밀 검사가 필요하면 다운로드 후 수동 검수를 권장합니다.

3. 업로드(재업로드) 기능은 현재 미구현입니다.
   향후 NCP Object Storage 연동으로 추가 예정입니다.
